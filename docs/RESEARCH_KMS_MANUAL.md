## 1пёҸвғЈвҖҜOverview

- **RSR** вҖ“ a structured markdown file under `plans/studies/<category>/study-<topic>-<status>.md`. It contains raw findings, a **Gaps / Open Questions** table, and a mandatory `decision_deadline` in the YAML frontвҖ‘matter.
- **ADR** вҖ“ a classic Architectural Decision Record (see https://adr.github.io). Once an RSR reaches **reviewed** and a decision is made, an ADR is created linking back to the RSR.

The `STUDIES_INDEX.md` (generated by `scripts/generate-index.js`) gives the Architecture Review Board a live dashboard with status emojis.

## 3пёҸвғЈвҖҜAdding New Research

1. **Create a raw markdown** in `plans/research/` (or directly write an RSR).
2. **Run the migration script**:
   ```bash
   npm run research:migrate
   ```
   The script will:
   - Read every file in `plans/research/`.
   - AutoвҖ‘assign a category (see the mapping table inside the script).
   - Generate an RSR with frontвҖ‘matter (`status: draft`, `decision_deadline` = today, etc.).
   - Move the original content into the `## Findings` section.
   - Log actions to `logs/researches/migration.log`.
3. **Edit the generated RSR** if you need to fineвҖ‘tune the `decision_deadline` or fill the **Gaps** table.
4. **Regenerate the index**:
   ```bash
   npm run docs:index
   ```
   The index is automatically added to the next commit via the preвҖ‘commit hook.

## 4пёҸвғЈвҖҜLifecycle & Statuses

| Emoji | Status     | Meaning                                             |
| ----- | ---------- | --------------------------------------------------- |
| вҡӘ    | `waiting`  | Placeholder created, no work started.               |
| рҹҹЎ    | `draft`    | Research is being collected.                        |
| рҹ”ө    | `reviewed` | Research is complete, ARB needs to make a decision. |
| рҹҹў    | `approved` | Decision made, ADR linked.                          |
| рҹҹӨ    | `archived` | Superseded or abandoned.                            |

The **GitHub Action** (`.github/workflows/study-checks.yml`) will warn the team when a `waiting` study is older than **21вҖҜdays** or a `reviewed` study older than **14вҖҜdays**.

## 5пёҸвғЈвҖҜGenerating the Index

The script `scripts/generate-index.js` walks `plans/studies/`, parses the YAML frontвҖ‘matter of each file and produces `plans/studies/index/STUDIES_INDEX.md`. It is automatically run:

- Manually via `npm run docs:index`.
- As part of the `preвҖ‘commit` hook (see `package.json`).

## 6пёҸвғЈвҖҜCreating an ADR

When a study reaches the **reviewed** state:

1. Open a new ADR using the ADR template (e.g., `adr-001-comgate-payment.md`).
2. Reference the RSR file in the **References** section.
3. Change the RSR status to `approved` and update `decision_deadline` if needed.
4. Commit both the ADR and the updated RSR.

## 8пёҸвғЈвҖҜTroubleshooting

- **Missing category** вҖ“ Edit `scripts/migrate-research-to-study.js` and add a mapping.
- **Index not updating** вҖ“ Ensure the preвҖ‘commit hook runs (`npm install` creates `.husky` scripts). Run `npm run docs:index` manually to verify.
- **Push fails** вҖ“ Verify `GIT_API_GRAINED_TOKEN` is set in your environment (run `echo $GIT_API_GRAINED_TOKEN`).
